# VoiceAid 测试报告（客户端）

- 版本：main 分支（截至 2026-01-25）
- 应用范围：Android 学前儿童汉语测评客户端（录音、结果统计、PDF 导出、历史同步）
- 报告类型：阶段性综合报告（含计划、设计、静态分析结论；未执行的项标注“未测”）
- 负责人：QA 架构

## 测试计划
### 测试策略与目标
- 策略：功能优先的风险驱动 + 基线单元测试补足 + 关键路径场景手工验证 + 弱网/权限/存储异常覆盖。
- 目标：
  - 确认核心业务链路“登录→模式选择→测评（A/E/RE/S/NWR/RG/PN/PST）→录音存储→结果→上传/下载→PDF 导出→历史”在支持端（Android 10/12/14）可用。
  - 验证录音、数据存储、网络上传、PDF 生成的正确性与稳定性。
  - 识别高风险缺陷（崩溃、数据丢失、上传失败无提示、权限受限等）。

### 测试范围
- In Scope：
  - 登录/注册、权限申请流程。
  - 测评各模块题目呈现、录音、结果计算与展示。
  - 音频本地存储与播放，索引文件维护（`utils/dataManager`）。
  - 历史记录（本地/云端）列表、详情、删除。
  - 数据上传/下载接口交互（`Netinteractutils`），含音频多线程上传。
  - PDF 导出（`PdfGenerator`）及文件保存。
  - 录音自检（`audiotest/Audiocheck`）。
- Out of Scope：
  - 服务器端模型评分准确性与可用性（仅验证客户端请求/解析）。
  - 第三方字体/素材版权合规。

### 测试环境
- 设备：小米 11(Android 14)、荣耀 30(Android 12)、Pixel 3a(Android 10)、Android Studio 模拟器 API 34。
- 构建：`./gradlew assembleDebug` 默认配置（targetSdk 34，minSdk 20，multidex 开启，混淆关闭）。
- 网络：Wi‑Fi 稳定、4G 弱网（延迟 200ms，丢包 10%）、离线模式。
- 账号与数据：本地调试登录（当前生产登录逻辑被注释），测试用 UID 预置；样例儿童信息与题库使用默认内置数据。

#### 测试结果综合分析及建议
- 最高风险场景：弱网上传/下载、音频缺失或未回调完成；PDF 生成依赖 SAF 如被后台杀死可能导致文件句柄丢失。
- 建议：
  1) 在上传完成前保持前台通知或阻止退出；
  2) 在结果页增加“上传状态”可见性与重试；
  3) PDF 生成前检查存储可用空间、捕获异常并提示用户。

#### 测试经验总结
- 业务耦合集中在 `evmenuactivity`，回归时需重点覆盖按钮跳转、参数传递（`format`, `Uid`, `isTest`）。

#### 测试结果综合分析及建议

#### 测试经验总结
- 在高 targetSdk (34) 下的存储/录音权限策略与早期设备差异大，系统测试必须覆盖 Android 10/12/14 的权限弹窗与存储路径可写性。

## 总结与发布建议
- 当前报告为“计划+设计+静态分析”版本，核心功能尚未执行实测。高风险集中在：
  - 弱网/并发上传未验证；
  - 录音/存储异常处理未验证；
  - PDF 生成与 SAF 流程可能被系统中断；
  - dataManager/Index 同步一致性未验证。
- 发布建议：暂不建议对外发布；先完成上述关键用例的实测与缺陷修复，再进行一次回归与性能抽测。
